# Design Context

## Users
Intelligence analysts and surveillance operators who build complex geospatial queries to filter device location records, cell tower pings, and events across multiple data sources. Power users who run 5-10 queries per session. They work in low-light operational environments and need precise, scannable interfaces.

## Brand Personality
**Operational. Precise. Restrained.**
The interface should feel like a professional intelligence tool — not flashy, not playful. Confident but quiet. Every element earns its place.

## Aesthetic Direction
- **Dark theme mandatory** — operational context requires low-light environments
- **Neutral-first**: The entire UI uses grey depth layering. No decorative color.
- **Yellow is reserved exclusively for the primary CTA** (Run Query button when active). Nothing else uses yellow in the query builder.
- **Red for danger only**: `t.danger` — exclusively for destructive actions (delete icon, confirmation bars)
- **Anti-references**: Dashboards with neon accents, gradient text, glassmorphism abuse, bouncy animations

## Design Principles

### 1. Yellow means "execute"
Yellow (#E0DD5B) appears only on the Run Query button when conditions are complete. No yellow on tabs, toggles, badges, icons, or hover states. This makes the CTA unmissable — if you see yellow, it means "this query is ready to run."

### 2. Grey communicates hierarchy through depth, not color
Active states use lighter greys, not accent colors. The grey scale — `bgBase` → `bgField` → `bgHover` → `bgRaised` — creates visual hierarchy through surface elevation alone. Active tabs use `bgHover` (subtle), toggle selections use `bgRaised`, hover states shift one step up.

### 3. Consistent 8px rhythm
All section padding (header, conditions, footer) uses `sp.sm` (8px). Side padding is 8px everywhere. Internal gaps use `sp.xs` (4px) or `sp.sm` (8px). No arbitrary values.

### 4. Animations serve transitions, not decoration
Collapse/expand uses orchestrated staging: content fades out first (100ms), then width shrinks (300ms expo). Expand reverses: width grows first, then content fades in with a slight delay. Chevron icons rotate rather than swap. All easing uses `easeOutExpo` for decisive feel.

### 5. Every interactive element has a 28px+ touch target
Buttons use `minWidth: 28, minHeight: 28` minimum. Rail buttons are 44x44. The only exception is the rail delete X (18x18) which is a hover-revealed secondary action.

## Design Tokens
All values come from the `t.*` token system (Arkem Design Tokens). Never hard-code colors. Spacing uses `sp.*` (4px base scale). Typography uses `type.*`. Motion uses `motion.*` with `easeOutExpo` for decisive interactions.

## Key Patterns
- Twin panel layout: 44px icon rail + content area
- Collapse/expand with orchestrated animation (content fades → width shrinks)
- Inline confirmation bars for destructive actions (neutral grey for reset, red-tinted for delete)
- Condition cards use 2-row stacked layout (Field on row 1, Operator + Value on row 2)
- Header: editable query name + action icons (save, reset, delete). No border-bottom — flows directly into conditions.
- Max 5 queries in rail. New queries start with one empty condition row.
- Source dropdown grouped by source groups with "Select Source…" placeholder.

## Input & Select Patterns
- **Custom chevron on all selects**: `appearance: none` with a custom SVG chevron (`10x6px`, stroke `#525252`) positioned `right 8px center`. Right padding `24px` to prevent text overlap. Never use native `appearance: auto` — it produces inconsistent chevrons across browsers.
- **Source selector**: Borderless pill with `bgField` background, `borderRadius: 6px`. No border, no focus ring. Blends into the header.
- **Condition selects (Field, Operator)**: Use `dtInputStyle` base with custom chevron override. Border from `borderSubtle`, `borderRadius: 6px`, `bgField` background.
- **Text inputs (Value)**: Use `dtInputStyle` with no chevron. Same border/radius as selects.
- **All inputs**: Consistent `4px 8px` padding. `colorScheme: dark` for native controls.
- **Focus state**: Condition inputs use `yellow500` border on focus (from shared `dtInputStyle`). Source selector has no visible focus indicator (borderless style).

## Bottom-Left Panel (BottomQueryBuilder) — Full Spec

### Role
This is the **primary query builder**. The top-left Conditions tab has been removed. This floating panel is the sole place for building and running queries.

### Positioning
- Absolute: `bottom: sp.sm`, `left: panelLeft`
- `panelLeft = panelCollapsed ? sp.sm : 320 + sp.sm + sp.sm` — tracks top-left panel collapse
- Open width: `min(320px, calc(100vw - 16px))`
- Collapsed width: `44px` (rail only)
- Max height: `460px` open, dynamic (query count × 44 + padding) when collapsed

### Surface
- **Panel**: `glassBg` / `glassBorder` — intentional. The glass style signals this is a floating overlay, distinct from the docked `bgBase` top-left panel.
- **Rail**: solid `bgBase` background — anchors the rail as the structural spine
- `borderRadius: sp.xs`, box shadow: `0 4px 24px rgba(0,0,0,0.5)` (stronger when expanded)
- `willChange: width` on panel container

### Closed State (Trigger Button)
- 44×44 icon button (`glassBg`, `glassBorder`)
- Search icon, `color: t.textSecondary`
- Hover: `bgHover` background
- Same absolute position as open panel

### Rail (44px)
| Element | Spec |
|---------|------|
| Database icon (top) | 44×44, decorative, `textSecondary`, `borderBottom: borderDark` |
| Query tab (inactive) | 44×44, transparent bg, `textSubtle`, `fontWeight: 500`, label `Q1`…`Q5` |
| Query tab (active) | 44×44, `bgHover` bg, `textPrimary`, `fontWeight: 600` |
| Query tab (hover) | `bgHover` bg (same as active — no over-elevation) |
| Condition count badge | `top: 3, right: 4`, `7px` font, `bg: textSecondary`, `color: textInverse`, `12px` tall, `borderRadius: 5` |
| Rail delete X | 18×18, `background: t.danger`, opacity 0 → 1 on row hover, top-right corner, X icon `8px`, `color: textPrimary`. **Instant delete, no confirmation.** |
| Divider | `width: 24, height: 1, background: borderDark` |
| New Query button | `BqRailBtn`, Plus icon, disabled at max 5 (tooltip shows limit) |
| Load button | `BqRailBtn`, FolderOpen icon |
| Collapse button | 44×44, ChevronLeft rotates 180° when collapsed, `easeOutExpo` |

### Header (Content Area)
- Background: `bgBase`, `minHeight: 44`, `paddingBottom: sp.sm`
- **Row 1** (`padding: 0 sp.sm`, `minHeight: 44`): Query name (editable, click-to-rename) + Save + Reset + Delete icons
  - Name: `type.subheading`, `color: textPrimary`, `cursor: text`, ellipsis overflow
  - Rename input: `borderBottom: 1px solid textPrimary`, no outer border, `borderRadius: 0`
  - Save/Reset: `QBarIcon`, `opacity: 0.3` + `disabled` when not applicable
  - Delete: `QBarIcon` with `Trash2 color={t.danger}` — always enabled, triggers confirmation bar
- **Row 2** (`padding: 0 sp.sm`, `marginTop: sp.xs`): Source select (flex: 1) + AND/OR toggle (`width: 72`)
- No `borderBottom` on header — content flows directly below

### Confirmation Bar (inline, `sectionFadeIn`)
| Action | Background | Border | Text color | Confirm button |
|--------|-----------|--------|-----------|---------------|
| Reset | `bgRaised` | `borderMuted` | `textPrimary` | `bg: textSecondary`, `color: textInverse` |
| Delete | `danger + "18"` | `danger + "40"` | `danger` | `bg: danger`, `color: "#fff"` |
- Cancel: `border: 1px solid borderMuted`, `color: textSecondary`
- Font: `11px`, `padding: 3px Xpx`, `borderRadius: sp.xs`

### Condition Area (`padding: sp.sm`, scrollable)
- Condition card: `bg: bgField`, `border: 1px solid borderDark`, `borderRadius: sp.xs`, `padding: sp.sm`
  - Row 1: Field select (fills width) + remove X (`28×28` min, X `11px`, `textSubtle`)
  - Row 2: Operator select + Value input
  - Link label between cards: `8px`, `fontWeight: 600`, `color: borderSubtle`, centered, `padding: 3px 0`
  - Disabled state: `opacity: 0.4`
- **+ Condition button**: dashed `borderMuted`, `minHeight: 36`, Filter icon `12px`
  - Hover: `bgHover` bg, `textPrimary` text, `borderSubtle` border

### Footer (`padding: sp.sm`, `borderTop: borderDark`)
- **Spatial toggle** (left): pill container `bg: bgField`, `border: borderDark`, `padding: 2`, `borderRadius: 4`
  - Options: "All" (Globe) + "View" (SquareDashed), `fontSize: 11`, `minHeight: 32`, `padding: sp.sm sp.md`
  - Active: `bgRaised` bg, `borderMuted` border, `textPrimary`, `fontWeight: 600`
  - Inactive: transparent, `textSubtle`, `fontWeight: 400`
- **Run Query** (right): `minHeight: 32`, `padding: sp.sm sp.md`, `fontSize: 12`, `fontWeight: 600`
  - Active: `yellow500` bg, `textInverse` text
  - Disabled: `bgHover` bg, `textSubtle` text, `cursor: default`

### Load Overlay (replaces content area)
- Header: `FolderOpen` icon + "Load Saved Query" label + close X
- List: each item `bg: bgField`, `border: borderDark`, hover → `borderMuted`
- Empty state: italic `textSubtle`, centered
- Metadata line: `9px`, `textSubtle`, condition count + saved date

### Collapse Animation (orchestrated)
- **Collapse**: content `opacity 0` + `translateX(-6px) scale(0.98)` in 100ms → width shrinks (`motion.fast`, `easeOutExpo`, 100ms delay)
- **Expand**: width grows first (`motion.slow`, `easeOutExpo`) → content fades in + translates in (`motion.medium`, `easeOutExpo`, 120ms delay)
- Rail `borderRight` fades to transparent when collapsed
