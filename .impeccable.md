# Design Context

## Users
Intelligence analysts and surveillance operators who build complex geospatial queries to filter device location records, cell tower pings, and events across multiple data sources. Power users who run 5-10 queries per session. They work in low-light operational environments and need precise, scannable interfaces.

## Brand Personality
**Operational. Precise. Restrained.**
The interface should feel like a professional intelligence tool — not flashy, not playful. Confident but quiet. Every element earns its place.

## Aesthetic Direction
- **Dark theme mandatory** — operational context requires low-light environments
- **Neutral-first**: The entire UI uses grey depth layering. No decorative color.
- **Yellow is reserved exclusively for the primary CTA** (Run Query button when active). Nothing else uses yellow in the query builder.
- **Red for danger only**: `t.danger` — exclusively for destructive actions (delete icon, confirmation bars)
- **Anti-references**: Dashboards with neon accents, gradient text, glassmorphism abuse, bouncy animations

## Design Principles

### 1. Yellow means "execute"
Yellow (#E0DD5B) appears only on the Run Query button when conditions are complete. No yellow on tabs, toggles, badges, icons, or hover states. This makes the CTA unmissable — if you see yellow, it means "this query is ready to run."

### 2. Grey communicates hierarchy through depth, not color
Active states use lighter greys, not accent colors. The grey scale — `bgBase` → `bgField` → `bgHover` → `bgRaised` — creates visual hierarchy through surface elevation alone. Active tabs use `bgHover` (subtle), toggle selections use `bgRaised`, hover states shift one step up.

### 3. Consistent 8px rhythm
All section padding (header, conditions, footer) uses `sp.sm` (8px). Side padding is 8px everywhere. Internal gaps use `sp.xs` (4px) or `sp.sm` (8px). No arbitrary values.

### 4. Animations serve transitions, not decoration
Collapse/expand uses orchestrated staging: content fades out first (100ms), then width shrinks (300ms expo). Expand reverses: width grows first, then content fades in with a slight delay. Chevron icons rotate rather than swap. All easing uses `easeOutExpo` for decisive feel.

### 5. Every interactive element has a 28px+ touch target
Buttons use `minWidth: 28, minHeight: 28` minimum. Rail buttons are 44x44. The only exception is the rail delete X (18x18) which is a hover-revealed secondary action.

## Color Usage (Query Builder)

| Element | Token | Rationale |
|---------|-------|-----------|
| Run Query (active) | `yellow500` bg | Only yellow in the component |
| Run Query (disabled) | `bgHover` bg | Lighter grey, clearly inactive |
| Active tab | `bgHover` bg, `textPrimary` text | Subtle elevation, barely visible |
| Inactive tab | transparent bg, `textSubtle` text | Recedes |
| Active toggle | `bgRaised` bg, `textPrimary` text | Elevated surface |
| Condition card | `bgField` bg | Mid-depth surface |
| Source selector | `bgField` bg, no border | Quiet pill, blends into header |
| + Condition hover | `bgHover` bg, `textPrimary` text | Neutral highlight |
| Delete icon | `danger` stroke, no fill | Red = destructive, ghost button |
| Save/Reset disabled | 0.3 opacity + `disabled` attr | Dimmed, not colored differently |
| Badges | `textSecondary` bg | Neutral, informational |
| Database icon | `textSecondary` | Decorative, recedes |

## Design Tokens
All values come from the `t.*` token system (Arkem Design Tokens). Never hard-code colors. Spacing uses `sp.*` (4px base scale). Typography uses `type.*`. Motion uses `motion.*` with `easeOutExpo` for decisive interactions.

## Key Patterns
- Twin panel layout (mirrors the top-left sidebar): 44px icon rail + content area
- Collapse/expand with orchestrated animation (content fades -> width shrinks)
- Inline confirmation bars for destructive actions (neutral grey for reset, red-tinted for delete)
- Condition cards use 2-row stacked layout (Field on row 1, Operator + Value on row 2)
- Header matches top-left QueryBar: editable name + action icons (save, reset, delete)
- No header border-bottom divider — header flows directly into conditions
- Max 5 queries in rail, new queries start with one empty condition row
- Source dropdown grouped by source groups with "Select Source..." placeholder

## Input & Select Patterns
- **Custom chevron on all selects**: `appearance: none` with a custom SVG chevron (`10x6px`, stroke `#525252`) positioned `right 8px center`. Right padding `24px` to prevent text overlap. Never use native `appearance: auto` — it produces inconsistent chevrons across browsers.
- **Source selector**: Borderless pill with `bgField` background, `borderRadius: 6px`. No border, no focus ring. Blends into the header.
- **Condition selects (Field, Operator)**: Use `dtInputStyle` base with custom chevron override. Border from `borderSubtle`, `borderRadius: 6px`, `bgField` background.
- **Text inputs (Value)**: Use `dtInputStyle` with no chevron. Same border/radius as selects.
- **All inputs**: Consistent `4px 8px` padding. `colorScheme: dark` for native controls.
- **Focus state**: Condition inputs use `yellow500` border on focus (from shared `dtInputStyle`). Source selector has no visible focus indicator (borderless style).
